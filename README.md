It will then starting to check your gold every 5 laps after lap#30. Find Battle will stop after you've reached certain gold amount or 80 laps, whichever comes first. Keep in mind that this is experimental feature, these numbers might change.<br>- <b>Use click for step</b>:<ul style="list-style-type: disc;"><li><b>Single step only</b>:&nbsp;if checked, the script will do click to move a single step instead of swiping. Move location will be determine by finding Rain's position on screen when exploration started. I recommend you check this for maximum compatibility. If the script can't find Rain it will turn this off automatically.</li><li><b>Always</b>: if swiping is not working for you then choose this.</li></ul>- <b>Lag Multiplier:</b>&nbsp;this setting is intended to use on slower devices or emulators, if your device lags, you might wanna set this to 2.0 or 2.5 or higher. I have successfully run Phantom Forest all night long on a 5 year old laptop with 2.5 lag multiplier.<br>- <b>Dungeon Finder</b>: as you can tell from the name: it will find all available dungeons on screen and give you a list to choose from. As of now, this option can be used to farm any dungeon, yes any dungeon including those in vortex.&nbsp;<br>- <b>Dim screen:</b> dim your screen while running.<br>- <b>Free Farm</b>: activate this when you're inside exploration, the script will keep going in your chosen direction and find battle until you stop it.<br>- <b>Colosseum Schedule</b>: if checked, it will do Colosseum battle every 2 hours and come back to your farming location.&nbsp;<br>Note: The following maps are not compatible with this feature, do not check this if you are planning to farm on these:<br><br><ul style="list-style-type: disc;"><li>Grandshelt Catacombs<br></li><li>Ballen Passage - Entrance<br></li><li>Maranda Coast<br></li><li>Vortex dungeons<br></li><li>to be continued... let me know so I can add</li></ul><br><br><b><font color="#dd1b00">How to define custom path:<br></font></b><br>This is what I originally planned for when starting this script. Here we go, create a new file called "<b><font color="#dd1b00">custom.txt</font></b>" and place it in the same folder of this script (which is ffbe.lua).<br>The general syntax for custom path is:<br><br><div class="quote no_header"><div class="quote_body"><b><font color="#dd1b00">map_name</font></b>:<font color="#006304">direction,steps</font><font color="#19007e" style="font-weight:bold;">|</font><font color="#006304">direction,steps</font><b>|</b><font color="#006304">direction,steps</font><b>|</b><font color="#006304">direction,steps</font><b>|</b>.......<b>|</b><font color="#006304">direction,steps</font><div class="quote_clear"></div></div></div><font color="#dd1b00"><b>map_name</b></font>: you name it with whatever you like, this name will show up on the selection list with a "custom_" prefix.<br><br><font color="#006304"><b>direction</b></font>:&nbsp;<br><ul style="list-style-type: circle;"><li>up, down, left, right</li><li>ul: go up-left (North West)</li><li>ur: go up-right (North East)</li><li>dl: go down-left</li><li>dr: down-right</li></ul><br><font color="#006304"><b>steps</b></font>: anything &lt;20 will be <i><b>single step</b></i> (trust me you don't need more than 20), anything above that will be <b><i>ms (milliseconds)</i></b>.<br> <br><b>Rule of thumb: one map per line<br>.</b>&nbsp;Make sure that you put all directions on a single line separated direction by a <b>|</b> (pipe).<br>&nbsp;<br>Direction example:<br><div class="quote no_header"><div class="quote_body"><b>up,5</b>: go up 5 steps<br><br><b>up,5000</b>: go up 5s<br><br><b>left,1000</b>: go left 1s<br><b>ul,1500:</b>&nbsp;go up-left 1.5s<div class="quote_clear"></div></div></div>Below is an example of my fulan path taken from the script:<br><br><div class="quote no_header"><div class="quote_body"><font face="courier new" size="2">fulan:up,2000|left,3000|up,1000|left,5|up,3000|right,1|left,1|down,3000|left,2000|up,1500|left,1500|up,3500|right,3000|up,2500|left,1000|up,500|left,2500|up,3000|left,2000|right,2000|down,3000|right,3000|down,2|right,3500|up,2000|right,4000|up,2000|left,3500|up,2000|left,1|up,5000|right,5000|up,6000|down,2|right,2000|up,3500|left,4000|right,4000|down,3500|<b><font color="#006304">battle,lr,400,left</font></b>|left,5000|up,500|left,1000|down,500|left,1000|down,2000|left,3500<br></font><div class="quote_clear"></div></div></div><br><font style="font-weight:bold;" color="#dd1b00">Special settings for custom path:<br></font>If you read my custom fulan path above you will notice the extra battle part. This setting is used for the <b>Find Battle</b> option, see below for syntax:<br><div class="quote no_header"><div class="quote_body">battle,[<b><font color="#006304">battle_direction</font></b>],[<b><font color="#ecbd00">gil_cap</font></b>],[<b><font color="#006304">exit_direction</font></b>]<br><div class="quote_clear"></div></div></div><ul style="list-style-type: disc;"><li><font color="#006304"><b>battle_direction</b></font>: either "<b>ud</b>" for up-down or "<b>lr</b>" for left-right</li><li><b><font color="#ecbd00">Gil cap</font>: </b>amount of gil to check, if &gt; this amount, the script will exit battle and continue exploration. How to get this? Either manually battle yourself or check here:&nbsp;<a href="http://redirect.viglink.com?key=bbb516d91daee20498798694a42dd559&amp;u=https%3A//exviuswiki.com/" rel="nofollow" target="_blank">exviuswiki.com/</a></li><li><b><font color="#006304">Exit direction</font></b>: direction to go next after done battle.</li></ul><br>Another setting is used to find moving direction for single steps (for <b>Use click to step</b> mode). When the exploration begin on some maps such as Phantom Forest or Maranda Coast, Rain's position might not be at the center of the screen, as a result you might need to use this to get Rain's position again. Put <b>findmove </b>at&nbsp;anyplace where you want the script to update Rain's position. Usually you only need to do this once and only need to do this if Rain's position is not at the center of the screen.&nbsp;See an example below for my Maranda Coast path:<br><br><div class="quote no_header"><div class="quote_body">down,8000|<b><font color="#006304">findmove</font></b>|up,13|left,5000|: go down 8s, then update Rain's position, then move up 13 steps, then left 5s<div class="quote_clear"></div></div></div><br><b><font color="#dd1b00">How to define a good path:<br></font></b><br>Below are few tips that I've learned from doing those exploration paths:<br>- First of all, try to minimize single steps if possible. Single steps take more time due to the fact that the script will wait and check for any battle encounter after each step.<br>- For any of your calculated ms, add 1000 to it.<br>- Minimum 500ms, don't go below that.<br>Also don't forget check out<b> <a target="_blank" href="http://ankulua.boards.net/post/2802/thread" rel="norewrite">this extremely handy trick</a></b> from <b><span itemscope="" itemtype="http://schema.org/Person"><a href="/user/2830" class="user-link js-user-link user-2830 group-0" itemprop="url" title="@eleck"><span itemprop="name">eleck</span></a></span>   .</b><br><font color="#dd1b00"><br><b>How to report a bug:</b></font><br><br>Please take a screenshot of your script config (like the first picture above), and advise what you were trying to do to produce that bug.&nbsp;<br><br><b>Demo:</b><br><iframe title="YouTube video player" width="560" height="349" src="//www.youtube.com/embed/TMOD8Y7alcM?wmode=transparent&amp;start=0" frameborder="0" allowfullscreen="1"></iframe><br><br>The farming details below was obtained from&nbsp;<a href="http://redirect.viglink.com?key=bbb516d91daee20498798694a42dd559&amp;u=https%3A//exviuswiki.com/Best_places_to_farm" rel="nofollow" target="_blank">exviuswiki.com/Best_places_to_farm</a>.
